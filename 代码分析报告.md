# ISO 55001审计工具代码结构与逻辑分析

## 1. 整体结构概述

ISO 55001审计工具是一个基于Streamlit构建的Web应用，用于评估组织对ISO 55001资产管理体系标准的合规程度。应用采用了模块化设计，清晰地划分了不同的功能区域。

### 1.1 主要组件

| 组件 | 功能描述 | 文件位置 |
|------|----------|----------|
| 数据库操作 | 评估结果的存储与加载 | app.py (19-78行) |
| 会话状态管理 | 用户评估过程的状态维护 | app.py (81-91行) |
| 问题加载 | 从YAML文件加载评估问题 | app.py (94-163行) |
| 分数计算 | 基于用户回答计算合规分数 | app.py (166-234行) |
| 图表生成 | 创建雷达图可视化评估结果 | app.py (237-299行) |
| PDF报告生成 | 生成详细的评估报告 | app.py (302-412行) |
| 主界面渲染 | 应用的用户界面与交互逻辑 | app.py (415-955行) |

### 4.3 使用数据模型

**当前问题**：使用字典表示复杂的数据结构，可读性和可维护性较差

**优化方案**：使用类来表示评估问题、响应等数据结构

```python
# 示例：使用类表示评估问题
class AssessmentQuestion:
    def __init__(self, q_id, question_type, description, sub_questions=None):
        self.q_id = q_id
        self.type = question_type
        self.description = description
        self.sub_questions = sub_questions or {}

    def get_description(self, lang='zh'):
        return get_translated_text(self.description, lang)

    def get_sub_questions(self, lang='zh'):
        return self.sub_questions.get(lang, [])
```

### 4.4 优化数据库设计

**当前问题**：数据库表结构简单，只存储了原始的响应数据

**优化方案**：增加更多的表和字段，提高数据存储的灵活性

```sql
-- 优化后的数据库表结构
CREATE TABLE IF NOT EXISTS assessments (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp TEXT,
    language TEXT,
    total_score REAL
);

CREATE TABLE IF NOT EXISTS assessment_responses (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    assessment_id INTEGER,
    section TEXT,
    question_id TEXT,
    response INTEGER,
    FOREIGN KEY (assessment_id) REFERENCES assessments(id)
);

CREATE TABLE IF NOT EXISTS assessment_sub_responses (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    assessment_id INTEGER,
    section TEXT,
    question_id TEXT,
    sub_question_index INTEGER,
    response BOOLEAN,
    FOREIGN KEY (assessment_id) REFERENCES assessments(id)
);
```
